Step 1 · Data access (Google Earth Engine)

GEE is the shortcut: petabytes of Landsat / Sentinel / ERA5 queried in the cloud — no downloading terabytes.

  • Sign up (free for research): earthengine.google.com/signup
  • Create a Google Cloud project and enable the Earth Engine API.
  • Authenticate in the terminal, then initialize in Python.
earthengine authenticate       # opens a browser, paste token back
import ee
ee.Initialize(project="your-gcp-project-id")

You get: the ability to pull any dataset in the brief with a few lines of Python.

Step 2 · Study area & time window

Pick ONE city and a cloud-free hot season — heat is the point.

city = ee.Geometry.Rectangle([77.5, 12.9, 77.7, 13.1])   # bbox lon/lat
start, end = "2023-04-01", "2023-06-30"              # hot pre-monsoon
Biggest time-saver: small area + one season first. Get the whole pipeline working end-to-end on a tiny region before scaling up.

Step 3 · Build the target — LST

This is your y — what the model predicts.

  • Landsat 8/9 Collection-2 L2 ships a surface-temp band ST_B10. Scale it: K = ST_B10·0.00341802 + 149.0, then °C = K − 273.15.
  • Mask clouds with the QA_PIXEL band; composite (median) into one clean image.
  • Optionally overlay ECOSTRESS LST for higher-res validation.
def scale_lst(img):
    lst = img.select("ST_B10").multiply(0.00341802).add(149.0).subtract(273.15)
    return lst.rename("LST")

col = (ee.ImageCollection("LANDSAT/LC08/C02/T1_L2")
       .filterBounds(city).filterDate(start, end)
       .filter(ee.Filter.lt("CLOUD_COVER", 20)))
lst = col.map(scale_lst).select("LST").median().clip(city)

You get: a clean LST raster in °C — your ground truth.

Step 5 · Baseline model (before the physics)

Always get a plain model working first, so you have something to beat — and it already completes brief Objectives 1 & 2.

from xgboost import XGBRegressor

model = XGBRegressor(n_estimators=400, max_depth=6, learning_rate=0.05)
model.fit(X_train, y_train)

Then run SHAP to quantify each driver's contribution — that's the driver attribution deliverable:

import shap
explainer   = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X)   # global + per-pixel attribution

You get: a working LST predictor + a ranked, explainable list of what drives the heat.

Step 6 · Add the physics (the core)

Move to a small PyTorch net so you can add a physics penalty to the loss. The surface energy balance:

Rn = H + LE + G — net radiation = sensible + latent + ground heat flux. The latent term (LE) is evaporative cooling: vegetation and water have high LE, which is why they cool. Penalize predictions that fight this.
import torch, torch.nn as nn

class LSTNet(nn.Module):
    def __init__(self, n):
        super().__init__()
        self.net = nn.Sequential(nn.Linear(n,64), nn.ReLU(),
                                 nn.Linear(64,64), nn.ReLU(),
                                 nn.Linear(64,1))
    def forward(self,x): return self.net(x).squeeze(-1)

def physics_residual(x, lst_pred, cols):
    # higher NDVI -> more LE cooling; higher albedo -> less absorbed Rn
    ndvi   = x[:, cols.index("NDVI")]
    albedo = x[:, cols.index("albedo")]
    expected = -(0.6*ndvi + 0.4*albedo)                 # physics-expected cooling
    anom     = (lst_pred - lst_pred.mean()) / (lst_pred.std()+1e-6)
    return ((anom - expected)**2).mean()

model, opt, lam = LSTNet(len(cols)), torch.optim.Adam(...), 0.2
for epoch in range(300):
    opt.zero_grad()
    pred = model(X_t)
    loss = ((pred - y_t)**2).mean() + lam*physics_residual(X_t, pred, cols)
    loss.backward(); opt.step()

What this buys you: the model can't learn "more trees → hotter" from noise. The physics term keeps the vegetation/albedo → cooling relationship sane — which is what makes Step 8's counterfactuals trustworthy.

Rigor upgrade (if time): replace the heuristic with real terms — Rn from net radiation, LE from an evapotranspiration model driven by NDVI, G as a fraction of Rn — and penalize Rn − (H + LE + G). Same structure, more defensible to judges.

Step 7 · Validate honestly

  • Spatial hold-out: train on some tiles, test on unseen ones (GroupKFold on tiles).
  • Temporal hold-out: train on one date, test another.
  • Report RMSE / MAE (accuracy) and a physics-consistency score.
  • Cross-check cooling estimates against InVEST Urban Cooling or SOLWEIG.

You get: evidence the model generalizes — the thing judges and planners actually care about.

Step 8 · Simulate cooling scenarios (the payoff)

This is why the physics mattered. To test "what if we green this area":

X_scn = X_hotspots.copy()
X_scn["NDVI"]   += 0.3       # add tree canopy
X_scn["albedo"] += 0.15      # reflective / cool roofs
lst_after = model(to_tensor(X_scn))
cooling   = lst_before - lst_after   # °C saved, per location

Then rank/optimize placements under a budget (greedy or integer program), weighting by population (GHSL) → intervention type + location + °C saved. That's Objective 4.
